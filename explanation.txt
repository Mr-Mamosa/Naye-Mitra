The behavior you're observing is expected. Here's a breakdown of why it happens:

1.  **Initial State**: When you first load the web application, the "Recent Consultations" list is empty because no queries have been made in the current session.

2.  **First Query**: When you submit your first query, two things happen:
    *   The backend processes your query and generates a response.
    *   A record of this interaction, including a new "case," is saved to a local database file (`data/audit_trail.db`).

3.  **Sidebar Update**: After the first query is successfully processed, the frontend automatically refreshes the "Recent Consultations" list. It fetches the newly created case from the backend, which reads it from the database, and then displays it in the sidebar.

**Your previous consultations are not lost.** They are stored in the `data/audit_trail.db` file in your project's `data` directory. The application is designed to show the consultations for the current user, and new consultations are added to the list as you make them.
